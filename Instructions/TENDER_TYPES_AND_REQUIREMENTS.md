## 0. TYPES OF USERS IN PROCUREMENT PROCESS

### 0.1 Primary Users (Procuring Entity Side)

| User Type | Role | Responsibilities | Authority Level |
|-----------|------|------------------|-----------------|
| **Procuring Entity (PE)** | Government agency/department making the purchase | Initiates procurement, manages entire process | Medium-High |
| **Employer/Purchaser** | Alternative term for Procuring Entity | Same as PE | Medium-High |
| **Approving Authority** | Authority that approves the award | Approves evaluation recommendations and contract award | High |
| **Head of PE** | Head of Procuring Entity | Overall accountability for procurement decisions | Highest |
| **Project Director/Manager** | Manages specific project | Oversees project procurement requirements | Medium |
| **Tender Opening Committee** | Opens and records tender submissions | Publicly opens bids, records details | Medium |
| **Evaluation Committee** | Evaluates technical and commercial proposals | Reviews bids, applies evaluation criteria | Medium-High |
| **Contract Management Committee** | Oversees contract execution | Monitors performance, handles variations | Medium |
| **Authorized Signatory** | Person authorized to sign documents | Signs tender documents, contracts | Low-Medium |
| **Authorized Officer** | For website posting | Posts advertisements, notices | Low-Medium |
| **Procurement Officer** | Handles day-to-day procurement activities | Prepares documents, coordinates with vendors | Medium |
| **Store/Warehouse Officer** | Receives and stores goods | Verifies delivery, maintains inventory | Low-Medium |
| **Accounts Officer** | Handles payments | Processes invoices, releases payments | Medium |
| **Audit Officer** | Reviews procurement compliance | Ensures compliance with rules | Medium |
| **Administrative Wings** | Other PE offices for circulation | Distributes information to other entities | Low |

### 0.2 Primary Users (Bidder Side)

| User Type | Role | Responsibilities | Authority Level |
|-----------|------|------------------|-----------------|
| **Quotationer** | Person/company submitting quotations in RFQ | Submits price quotations for low-value procurement | Low-Medium |
| **Tenderer** | Person/company submitting tenders | Submits technical and commercial proposals | Medium-High |
| **Supplier** | For goods procurement | Supplies goods as per contract | Medium |
| **Contractor** | For works procurement | Executes construction/works as per contract | Medium-High |
| **Service Provider** | For services procurement | Provides services as per contract | Medium |
| **Sole Proprietor** | Individual business owner | Manages business entity | High |
| **Managing Director** | Company executive | Authorized representative for company | Highest |
| **Authorized Partner** | Partnership firm representative | Authorized to act on behalf of partnership | High |
| **Chairman/CEO** | Top executive | Ultimate authority for company decisions | Highest |

### 0.3 Supporting Users

| User Type | Role | Responsibilities | Authority Level |
|-----------|------|------------------|-----------------|
| **Bank** | Financial institution | Issues securities, guarantees, letters of credit | High (Financial) |
| **JV Partners** | Joint Venture partners in tendering | Collaborate on large contracts | High (Consensus) |
| **Subcontractors** | Parties contracted by main bidder | Execute specific portions of work | Medium |
| **Manufacturer** | Original equipment manufacturer | Supplies goods, provides authorization | Medium-High |
| **Consultants** | Technical advisors | Prepare specifications, provide technical advice | Medium |
| **Audit/Inspection Agencies** | Quality control | Inspect goods/works/services | Medium |
| **BPPA** | Bangladesh Public Procurement Authority | Regulates procurement process | High (Regulatory) |
| **CPTU** | Central Procurement Technical Unit | Manages e-GP system | High (Technical) |

## 0. TYPES OF PURCHASE METHODS

### 0.1 Request for Quotation (RFQ) Method

| Code | Applicable For | Value Limit | Key Features |
|------|---------------|-------------|--------------|
| **PG1** | Goods | Up to BDT 8 Lac (0.8 million) | No tender security, 3-7 days submission, unit-rate basis |
| **PW1** | Works | Up to BDT 15 Lac (1.5 million) | No tender security, unit-rate or lump-sum, simple works |

**Characteristics:**
- Simplified process
- No newspaper advertisement (website/notice board only)
- Free document issuance
- No tender security or performance security required
- Minimum 3 days for submission (maximum 7 days)
- No public opening ceremony
- Simple L1 (lowest price) evaluation
- Purchase Order instead of formal contract

**When to Use:**
- Low-value, readily available standard goods
- Simple works with clear scope
- Off-the-shelf items
- Emergency situations requiring quick procurement

### 0.2 Open Tendering Method (OTM)

| Code | Applicable For | Value Limit | Key Features |
|------|---------------|-------------|--------------|
| **PG3** | Goods | Above BDT 50 Lac | Full tender process, national competition |
| **PG4** | Goods | Any value | International competition |
| **PW3** | Works | Above BDT 5.00 Crore | Large works, no pre-qualification |
| **PPS2** | Services (Outsourcing Personnel) | Any value | Manpower hiring |
| **PPS3** | Services (Other Physical Services) | Any value | General physical services |

**Characteristics:**
- Competitive bidding open to all qualified bidders
- Newspaper advertisement required (national daily)
- Tender security required
- Performance security required
- Detailed evaluation criteria
- Public bid opening ceremony
- Formal contract signing
- Pre-qualification optional (except PW3)

**When to Use:**
- Medium to high-value procurements
- When maximum competition is desired
- For complex requirements
- When transparency is critical

### 0.3 Limited Tendering Method (LTM)

| Code | Applicable For | Value Limit | Key Features |
|------|---------------|-------------|--------------|
| **PG2** | Goods | Up to BDT 50 Lac | Restricted to qualified suppliers |
| **PPS2** | Services (Outsourcing Personnel) | Any value | Restricted to pre-qualified service providers |
| **PPS3** | Services (Other Physical Services) | Any value | Restricted to pre-qualified service providers |

**Characteristics:**
- Invitation sent to selected bidders only
- Pre-qualification criteria established
- Shorter procurement timeline
- Reduced competition but more control
- Same formalities as Open Tendering

**When to Use:**
- Specialized goods/services with limited suppliers
- When specific expertise is required
- For repeat procurements with known reliable suppliers
- When time constraints exist

### 0.4 Direct Procurement Method

| Code | Applicable For | Value Limit | Key Features |
|------|---------------|-------------|--------------|
| **PG9A** | Goods | Any value | Single source, exceptional circumstances |
| **PPS6** | Services | Any value | Single source, exceptional circumstances |

**Characteristics:**
- Single source procurement
- No competitive bidding
- Requires justification
- Approval from higher authority
- Simplified documentation
- Price verification required

**When to Use:**
- Only one supplier available (monopoly)
- Proprietary items with no substitutes
- Emergency situations
- National security considerations
- Continuation of specialized services
- Research and development contracts

### 0.5 Turnkey Contract Method

| Code | Applicable For | Value Limit | Key Features |
|------|---------------|-------------|--------------|
| **PG5A** | Plant & Equipment | Any value | Design, supply, installation, commissioning |

**Characteristics:**
- Complete responsibility from design to commissioning
- Two-stage technical and commercial evaluation
- Mandatory pre-bid meetings
- Mandatory site visits
- Detailed technical specifications
- Performance-based evaluation
- Higher security requirements

**When to Use:**
- Supply and installation of complex plants/equipment
- When single-point responsibility is desired
- For turnkey projects requiring design integration
- When technical capability is more important than price

## Document Requirements by Tender Type

### PG1: RFQ for Goods (up to 8 Lac)

#### Key Characteristics
- ❌ **NO Tender Security required** (Rule 91(6))
- ❌ **NO Performance Security required**
- ✅ Free document issuance (no cost to vendors)
- ✅ Minimum 3 days, maximum 7 days submission time
- ✅ No newspaper advertisement (website, notice board, nearby entities only)
- ✅ Price basis: **Unit-Rate**
- ✅ Splitting procurement NOT permitted

#### Documents Required
1. Quotation Submission Letter
2. Price Schedule (Goods + Delivery)
3. Specifications Compliance Sheet
4. Company/Individual profile
5. Valid Trade License
6. TIN Certificate

#### Evaluation
- Simple L1 (Lowest evaluated price)
- Compliance check only
- No complex scoring

#### Contract Award
- Purchase Order issued directly
- No formal contract signing ceremony

---

### PG2: Tender for Goods (up to 50 Lac)

#### Key Characteristics
- ✅ **Tender Security required** (Pay Order/Bank Draft/Bank Guarantee)
- ✅ **Performance Security required**
- ✅ Validity: 28 days beyond tender validity
- ✅ Price basis: **Unit-Rate or Lump-Sum**
- ✅ Newspaper advertisement required
- ✅ Two-Envelope system optional

#### 8 Standard Sections
1. **Instructions to Tenderers (ITT)** - FIXED, cannot modify
2. **Tender Data Sheet (TDS)** - Customizable per tender
3. **General Conditions of Contract (GCC)** - FIXED, cannot modify
4. **Particular Conditions of Contract (PCC)** - Customizable
5. **Tender & Contract Forms** (Forms PG2-1 to PG2-11)
6. **Schedule of Requirements**
7. **Technical Specifications**
8. **Drawings** (if applicable)

#### Forms Required (11 Forms)

**Tender Forms (PG2-1 to PG2-7):**
- **PG2-1:** Tender Submission Letter
- **PG2-1A:** Letter of Authorization
- **PG2-2:** Tenderer Information Sheet
- **PG2-3A:** Price Schedule for Goods
- **PG2-3B:** Price Schedule for Related Services
- **PG2-4:** Specifications Submission and Compliance Sheet
- **PG2-5:** Manufacturer's Authorization Letter (if applicable)
- **PG2-6:** Bank Guarantee for Tender Security
- **PG2-7:** Bank's Letter of Commitment for Line of Credit (if required)

**Contract Forms (PG2-8 to PG2-11):**
- **PG2-8:** Notification of Award
- **PG2-9:** Contract Agreement
- **PG2-10:** Bank Guarantee for Performance Security
- **PG2-11:** Contract Amendment

#### Evaluation Criteria
- Technical compliance
- Price evaluation
- Delivery schedule
- Payment terms
- Warranty conditions

---

### PG3: Tender for Goods (above 50 Lac, National)

#### Key Characteristics
- ✅ **Tender Security required**
- ✅ **Performance Security required**
- ✅ **Two-Envelope system** possible (Technical + Commercial)
- ✅ Detailed qualification requirements
- ✅ Mandatory newspaper advertisement (national daily)
- ✅ Minimum tender validity: 90 days (default)

#### Same 8 Sections as PG2 with Enhanced Requirements

#### Additional Documents Required
1. **Detailed Company Profile:**
   - Organizational structure
   - Key management team
   - Annual turnover (last 3 years)
   - Net worth statement

2. **Technical Capacity:**
   - Manufacturing facilities (if manufacturer)
   - Quality control systems
   - ISO/Quality certifications
   - Testing laboratory details

3. **Financial Capacity:**
   - Audited financial statements (last 3 years)
   - Bank solvency certificate (specific amount)
   - Letter of credit commitment
   - Liquid assets proof

4. **Experience Certificates:**
   - Similar nature contracts (last 5 years)
   - Value and scope of past projects
   - Client references with contact details
   - Performance certificates

5. **Legal Documents:**
   - Litigation history declaration
   - Tax clearance certificate
   - VAT registration
   - Trade license
   - Partnership deed / Memorandum & Articles (if company)

6. **Technical Documents:**
   - Detailed technical specifications compliance
   - Test reports / certificates
   - Catalogue / brochures
   - Samples (if required)

#### Evaluation
- Pre-qualification criteria
- Technical evaluation (weighted scoring)
- Commercial evaluation
- Combined scoring or L1 basis

- ✅ Mandatory pre-bid meeting

### PG9A: Direct Procurement for Goods

#### When Allowed (Exceptional Circumstances Only)
1. Single source available (monopoly)
2. Proprietary items with no substitutes
3. Emergency situations
4. National security
5. Continuation of services from same supplier

#### Simplified Structure (4 Sections)

#### Key Features
- No competitive bidding
- Price verification from multiple sources
- Justification documentation required
- Approval from higher authority (usually Ministry or Division)

#### Documents Required

   - Justification for direct procurement
   - Market survey report (price comparison)

   - Price verification documents
   - Approval from competent authority

2. **From Supplier:**
   - Price quotation
   - Technical compliance statement
   - Delivery schedule
   - Payment terms
   - Warranty conditions

### PW1: RFQ for Works (up to 15 Lac)

#### Key Characteristics
- ❌ **NO Tender Security required**
- ❌ **NO Performance Security required**
- ✅ Free document issuance
- ✅ Minimum 3 days, maximum 7 days submission
- ✅ No newspaper advertisement
- ✅ Price basis: **Unit-Rate or Lump-Sum**
- ✅ Retention money: 5% of contract price
- ✅ Defect liability period: 6-12 months

#### Documents Required
1. Quotation Submission Letter
2. Bill of Quantities (Unit-Rate or Lump-Sum)
3. Specifications Compliance Sheet
4. Company/Individual profile
5. Valid Trade License
6. TIN Certificate
7. Financial Solvency Certificate (30-50% of estimated cost)

#### Evaluation
- Simple L1 evaluation
- Compliance check
- At least 3 responsive quotations required
- SLT (Significantly Low-price Tender) check

#### Payment Terms
- Unit-Rate basis: Progressive payments based on measured work
- Lump-sum basis: Payments in 3 installments (30%, 75%, 100%)
- 5% retention money deducted from each invoice
- Retention money released after defect liability period

## Common Document Requirements (All Tender Types)

### Mandatory for ALL Tenderers

#### Legal Documents
1. **Valid Trade License**
   - Current and valid
   - Relevant to the nature of business
   - Certified true copy

2. **Tax Identification Number (TIN) Certificate**
   - Valid TIN certificate
   - Tax clearance certificate
   - Or acknowledgement slip of income tax return

3. **VAT Registration Certificate**
   - Valid VAT registration
   - Business Identification Number (BIN)
   - Current and valid

4. **Financial Solvency Certificate**
   - From a scheduled bank
   - Shows liquid assets
   - Amount: 30-70% of estimated cost (varies by tender type)

5. **Partnership Deed / Memorandum & Articles**
   - If partnership firm
   - If company (registered)
   - Certified copy

#### Financial Documents
1. **Audited Financial Statements**
   - Last 3-5 years (varies by tender type)
   - Balance sheet
   - Income statement
   - Cash flow statement
   - Auditor's report

2. **Bank Statements**
   - Last 6-12 months
   - Shows liquidity

3. **Credit Line Commitment**
   - Letter from bank
   - Shows available credit
   - Amount: varies by tender type

#### Experience Documents
1. **Experience Certificates**
   - From previous clients
   - Similar nature of work
   - Value and scope
   - Time period covered

2. **Completed Contracts List**
   - Last 5-10 years (varies by tender type)
   - Contract number
   - Client name
   - Contract value
   - Completion date

3. **Performance Certificates**
   - From previous clients
   - Shows satisfactory performance
   - Completion confirmation

#### Organizational Documents
1. **Company Profile**
   - Organizational structure
   - Key management team
   - Office locations
   - Facilities available
   - Major clients

2. **Personnel Information**
   - Key personnel qualifications
   - Experience
   - Roles and responsibilities
   - Commitment letters

3. **Equipment and Machinery List**
   - Owned equipment
   - Hired equipment availability
   - Capacity and capability

#### Declarations
1. **No Corruption Declaration**
   - Not involved in corrupt practices
   - Not debarred by government
   - Not suspended

2. **No Litigation Declaration**
   - No pending litigation
   - No insolvency
   - Not bankrupt

3. **Not Government Owned Entity Declaration**
   - Or confirmation of compliance if government owned

4. **Single Tenderer Declaration**
   - Not submitting more than one tender
   - Not participating through different names

5. **Nationality Declaration**
   - Eligible country nationals
   - For international tenders

### Additional for Limited Tendering
1. Pre-qualification documents
2. Selection criteria compliance
3. Invitation acceptance

### Additional for International Tenders
1. Import/export licenses
2. Foreign exchange clearance
3. Certificate of origin
4. International quality certificates
5. Manufacturer's authorization (if dealer)

## Complete Procurement Process Flow

### Phase 1: Planning and Preparation

#### Step 1.1: Need Identification
**Responsible:** Procuring Entity / Project Manager
- Identify procurement need
- Define requirements clearly
- Prepare procurement plan
- Budget allocation
- Approve procurement plan

**Information Required:**
- Description of goods/works/services
- Quantity/Volume
- Technical specifications
- Quality standards
- Delivery/Completion requirements
- Budget allocation

**Fields/Forms:**
- Procurement Plan Form
- Needs Assessment Report
- Budget Approval Document

#### Step 1.2: Tender Type Selection
**Responsible:** Procuring Entity / Approving Authority
- Determine appropriate tender type based on:
  - Value of procurement
  - Nature of goods/works/services
  - Complexity
  - Availability of suppliers
  - Time constraints
  - Emergency situations

**Decision Matrix:**
```
If value ≤ 8 Lac (goods) or ≤ 15 Lac (works) → Use RFQ (PG1/PW1)
If value ≤ 50 Lac (goods) → Use PG2 (Open/Limited)
If value > 50 Lac (goods) → Use PG3 (Open Tender)
If international competition required → Use PG4
If plant & equipment → Use PG5A (Turnkey)
If single source → Use PG9A/PPS6 (with justification)
If value > 5 Crore (works) → Use PW3
If outsourcing personnel → Use PPS2
If other physical services → Use PPS3
```

**Fields/Forms:**
- Tender Type Selection Form
- Justification Document (for Direct Procurement)

#### Step 1.3: Tender Document Preparation
**Responsible:** Procurement Officer / Procuring Entity
- Select appropriate standard tender document (STD)
- Complete Tender Data Sheet (TDS)
- Prepare Technical Specifications
- Prepare Bill of Quantities / Schedule of Requirements
- Prepare Particular Conditions of Contract (PCC)
- Set qualification criteria
- Set evaluation criteria
- Calculate tender security amount
- Set tender validity period
- Determine submission deadline

**Information Required:**
- Name and address of Procuring Entity
- RFQ/Tender number
- Date of issue
- Name and description of goods/works/services

## Summary: When to Use Which Tender Type

### Quick Decision Tree

```
┌─ GOODS PROCUREMENT ─┐
│                     │
├─ Value ≤ 8 Lac     → Use PG1 (RFQ)
├─ Value 8-50 Lac    → Use PG2 (Limited/Open Tender)
├─ Value > 50 Lac    → Use PG3 (Open Tender National)
├─ International     → Use PG4 (any value)
├─ Plant Install     → Use PG5A (Turnkey)
└─ Single Source     → Use PG9A (Direct) [with justification]

┌─ WORKS PROCUREMENT ─┐
│                     │
├─ Value ≤ 15 Lac    → Use PW1 (RFQ)
├─ Value > 5 Crore   → Use PW3 (Open Tender)
└─ Single Source     → Direct Procurement [with justification]

┌─ SERVICES PROCUREMENT ─┐
│                        │
├─ Outsourcing Personnel → Use PPS2
├─ Other Physical Services → Use PPS3
└─ Single Source         → Use PPS6 (Direct) [with justification]
```

### Selection by Value (BDT)

| Value Range | Goods | Works | Services |
|-------------|-------|-------|----------|
| **Up to 8 Lac** | PG1 (RFQ) | — | — |
| **8 Lac - 15 Lac** | PG2 | PW1 (RFQ) | — |
| **15 Lac - 50 Lac** | PG2 | — | — |
| **50 Lac - 5 Crore** | PG3 | — | PPS2/PPS3 |
| **Above 5 Crore** | PG3/PG4 | PW3 | PPS2/PPS3 |
| **Any Value (Special)** | PG5A, PG9A | — | PPS6 |

### Selection by Scope

| Requirement | Recommended Method |
|-------------|-------------------|
| Standard off-the-shelf goods | PG1 (RFQ) |
| Custom/specialized goods | PG2/PG3 |
| International sourcing | PG4 |
| Complete factory/plant setup | PG5A (Turnkey) |
| Simple construction | PW1 (RFQ) |
| Major construction project | PW3 |
| Hiring security guards/staff | PPS2 |
| IT/facility management services | PPS3 |
| Emergency procurement | PG9A/PPS6 (Direct) |
| Proprietary item/service | PG9A/PPS6 (Direct) |

## Implementation Guidelines for RFQ Platform

### Tender Type Selection Logic

```javascript
function determineTenderType(procurement) {
  const { type, value, scope, isInternational, isEmergency, isSingleSource } = procurement;
  
  // Direct procurement for emergencies or single source
  if (isEmergency || isSingleSource) {
    if (type === 'GOODS') return 'PG9A';
    if (type === 'SERVICES') return 'PPS6';
  }
  
  // Goods procurement
  if (type === 'GOODS') {
    if (scope === 'TURNKEY') return 'PG5A';
    if (isInternational) return 'PG4';
    if (value <= 800000) return 'PG1';        // 8 Lac
    if (value <= 5000000) return 'PG2';       // 50 Lac
    return 'PG3';                             // Above 50 Lac
  }
  
  // Works procurement
  if (type === 'WORKS') {
    if (value <= 1500000) return 'PW1';       // 15 Lac
    if (value > 50000000) return 'PW3';       // Above 5 Crore
  }
  
  // Services procurement
  if (type === 'SERVICES') {
    if (scope === 'OUTSOURCING_PERSONNEL') return 'PPS2';
    return 'PPS3';
  }
}
```
