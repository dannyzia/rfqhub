# Bangladesh e-GP Tender Types - User Guide

## Overview

The RFQ Buddy platform now supports all 14 official Bangladesh e-GP tender types as per PPR 2025. This guide explains how to use the tender type system.

---

## For Buyers (Procuring Entities)

### Creating a Tender with the Right Type

#### 1. Navigate to Create Tender
- Click "Create New Tender" from the dashboard
- Fill in basic tender information (title, description)

#### 2. Enter Basic Information
- **Title**: Brief description of procurement
- **Procurement Type**: Choose Goods, Works, or Services
- **Estimated Cost**: Your preliminary budget

#### 3. Select Tender Type (Smart Wizard)
Click "Select Tender Type" to launch the interactive wizard.

**Step 1: Confirm Procurement Type**
- **Goods**: Equipment, supplies, materials, stationery
- **Works**: Construction, infrastructure, repairs, maintenance
- **Services**: Consulting, outsourcing, IT services, training

**Step 2: Enter Estimated Value (BDT)**
- Enter your exact budget in Bangladeshi Taka
- The system shows formatted amount (Lac/Crore)
- Examples:
  - 300,000 → 3.00 Lac
  - 5,000,000 → 50.00 Lac
  - 100,000,000 → 10.00 Crore

**Step 3: Additional Options (Optional)**
Select any that apply to your procurement:
- **International Competition**: Allow foreign bidders to participate
- **Emergency Procurement**: For disaster/crisis/urgent situations
- **Single Source**: Only one supplier available (requires justification)
- **Turnkey Contract** (Goods only): Supply + Install + Commission model
- **Outsourcing Personnel** (Services only): Guards, cleaners, drivers, etc.

#### 4. Review Suggestions
- System shows recommended tender types with confidence scores
- **Green highlight** = best match for your situation
- **Yellow highlight** = acceptable alternative
- Click any suggestion to view full details and requirements
- Select the most appropriate tender type

#### 5. Review Auto-Calculated Values
- **Bid Security %**: Automatically calculated (e.g., 2% for PG2)
- **Performance Security %**: Shown for reference
- **Minimum Submission Period**: Auto-validated (e.g., 15 days for PG2)
- **Number of Forms Required**: Type-specific documentation count

#### 6. Publish Tender
- System enforces all PPR 2025 requirements
- Document checklist auto-generated based on tender type
- Bid submission rules activated
- Cannot proceed if validation fails

## Understanding Tender Types

### Goods Procurement (PG Series)

**PG1 - Request for Quotation (RFQ) for Goods**
- **Value Range**: Up to BDT 8,00,000
- **Best For**: Small purchases, office supplies, simple goods
- **Bid Security**: None required
- **Minimum Submission**: 3-7 days
- **Documents Required**: 6 (simplified)
- **Note**: Fastest process, minimal documentation

**PG2 - Open Tendering Method (Goods)**
- **Value Range**: BDT 8,00,000 - 50,00,000
- **Best For**: Medium purchases, equipment, materials
- **Bid Security**: 2% of tender value
- **Minimum Submission**: 15-30 days
- **Documents Required**: 18 forms
- **Note**: Standard procurement process

**PG3 - Open Tendering Method (Large Goods)**
- **Value Range**: Above BDT 50,00,000
- **Best For**: Large equipment, bulk purchases, vehicles
- **Bid Security**: 2% of tender value
- **Minimum Submission**: 30+ days
- **Documents Required**: 20+ forms
- **Note**: Enhanced requirements, newspaper ads required

**PG4 - Limited Tendering Method**
- **Value Range**: Any value (with justification)
- **Best For**: Specialized goods with limited suppliers
- **Bid Security**: 2%
- **Minimum Submission**: 15-21 days
- **Documents Required**: 18 forms
- **Note**: Requires written justification for limited competition

**PG5A - International Open Tendering**
- **Value Range**: Large scale (50+ Lac recommended)
- **Best For**: Goods needing international competition
- **Bid Security**: 2-3%
- **Minimum Submission**: 45+ days
- **Documents Required**: 20+ forms (multi-language)
- **Note**: Two-envelope system required

**PG9A - Direct Procurement**
- **Value Range**: Any value
- **Best For**: Emergency situations, single supplier
- **Bid Security**: May be waived
- **Minimum Submission**: 1-2 days
- **Documents Required**: Simplified + justification
- **Note**: Requires special approval and documented justification

### Works Procurement (PW Series)

**PW1 - RFQ for Works (Small)**
- **Value Range**: Up to BDT 15,00,000
- **Best For**: Minor repairs, small construction
- **Bid Security**: None
- **Minimum Submission**: 7 days
- **Documents Required**: 5-6
- **Note**: Similar to PG1 but for construction

**PW2 - Open Tendering (Works)**
- **Value Range**: BDT 15,00,000 - 5,00,00,000
- **Best For**: Standard construction projects
- **Bid Security**: 5%
- **Minimum Submission**: 21-30 days
- **Documents Required**: 15+ forms

**PW3 - Open Tendering (Large Works)**
- **Value Range**: Above BDT 5,00,00,000
- **Best For**: Major infrastructure projects
- **Bid Security**: 5%
- **Minimum Submission**: 45+ days
- **Documents Required**: 25+ forms
- **Note**: Pre-qualification may be required, two-envelope system

### Services Procurement (PPS Series)

**PPS2 - Outsourcing of Personnel**
- **Best For**: Hiring guards, cleaners, drivers, maintenance staff
- **Selection Method**: Quality-Cost-Based (60% quality, 40% cost)
- **Bid Security**: 2%
- **Contract Type**: Performance-based, usually annual
- **Documents Required**: Supplier credentials, samples

**PPS3 - Least-Cost Selection**
- **Best For**: Standard services (printing, transportation, training)
- **Selection Method**: Technical qualification → Lowest cost wins
- **Bid Security**: 2%
- **Minimum Submission**: 15-30 days
- **Documents Required**: Technical capability, cost quote

**PPS4 - Quality-Cost-Based Selection**
- **Best For**: Complex services (consulting, IT)
- **Selection Method**: Quality (70%) + Cost (30%)
- **Bid Security**: 2%
- **Documents Required**: Experience, qualifications, proposal

**PPS5 - Risk-Based Selection**
- **Best For**: High-risk projects needing experienced vendors
- **Selection Method**: Technical capability weighted heavily
- **Bid Security**: 2%

**PPS6 - Direct Procurement (Services)**
- **Best For**: Emergency services, single available specialist
- **Selection Method**: Direct award
- **Bid Security**: May be waived
- **Documents Required**: Justification + service provider details

## For Vendors (Bidders)

### Preparing Your Bid

#### 1. View Tender Details
- Click on any published tender you're interested in
- Review tender type, value, and timeline
- Check all requirements carefully

#### 2. Review Document Checklist
- Click "View Required Documents"
- **Documents marked ORANGE** = Mandatory (must submit)
- **Documents marked BLUE** = Optional (strengthen your bid)
- Note the file size limits and formats

**Document Checklist Example:**
```
Required Documents for PG1:
 ✓ MANDATORY - Technical Bid Sheet
 ✓ MANDATORY - Price Bid
 ✓ MANDATORY - Company Registration Certificate
   OPTIONAL - Company Profile
   OPTIONAL - Previous Similar Projects
```

#### 3. Prepare Documents
- Collect all mandatory documents first
- Ensure proper format (PDF recommended for all)
- Keep file sizes ≤ 10MB each
- Organize by category (technical, commercial)

#### 4. Upload Documents
- Click "Upload Document" for each requirement
- Select file from your computer
- System shows upload progress
- Check mark appears when uploaded successfully

**Supported File Formats:**
- Documents: PDF, DOCX (Word), XLS (Excel)
- Images: JPG, PNG
- Maximum size: 10MB per file

#### 5. Validation Before Submission
The system automatically checks:
- ✓ All mandatory documents are uploaded
- ✓ Bid security amount matches requirement
- ✓ Submission within deadline
- ✓ File formats are correct

**You CANNOT submit bid until all checks pass.**

#### 6. Submit Bid
- Click "Submit Bid" when checklist shows complete (100%)
- Review final summary
- Confirm submission
- Receive confirmation email with receipt number

## Common Questions & Answers

### For Buyers

**Q: What if I select the wrong tender type?**
A: You can change it anytime before publishing. After publishing, contact System Admin (it requires approval to change).

**Q: Why can't I use PG1 for a 15 Lac purchase?**
A: PG1 is limited to BDT 8 Lac per PPR 2025. For 15 Lac, use PG2. The system will show an error if you try to set it incorrectly.

**Q: What is "two-envelope system"?**
A: Technical bid and commercial bid are submitted together but evaluated separately. Evaluators open technical envelopes first, then commercial bids only for qualified bidders.

**Q: Do I need to calculate bid security?**
A: No, the system auto-calculates based on tender type. For example, PG2 = 2% of tender value. You just arrange payment before tender closure.

**Q: Can I change the value after setting tender type?**
A: Yes, but changing the value might suggest a different tender type. The system will alert you if the new value is outside the type's range.

**Q: What if my budget is exactly 8 Lac?**
A: Use PG2 (8-50 Lac range). PG1 is strictly below 8 Lac.

### For Vendors

**Q: Can I upload documents after submitting my bid?**
A: No, all documents must be uploaded before bid submission. Plan ahead and give yourself extra time.

**Q: What if I'm missing one non-mandatory document?**
A: You can still submit, but mandatory documents MUST be present. Adding optional documents strengthens your bid.

**Q: How long do I have to upload documents?**
A: Until the tender deadline. It varies by type:
- PG1: 3-7 days
- PG2: 15-30 days
- PG3: 30+ days

**Q: Can I replace an uploaded document?**
A: Yes, delete the old one and upload a new version. Just make sure to do it before the deadline.

**Q: What file format should I use?**
A: PDF is safest for all documents (preserves formatting). DOC/DOCX/XLS are also accepted.

**Q: Is there a maximum file size for uploads?**
A: Yes, 10MB per file. If your file is larger, compress it or split into multiple parts.

## System Rules & Compliance

This system implements Bangladesh government regulations:
- **Public Procurement Act (PPA) 2006**
- **Public Procurement Rules (PPR) 2025**
- **Bangladesh e-GP Guidelines (Revised) 2025**

All tender types, value thresholds, security percentages, and submission periods are mandated by law. The system enforces these automatically.
