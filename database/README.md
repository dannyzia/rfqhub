The main schema file is located at:
```
RFQ_Buddy/Instructions/rfq_tendering_platform_schema_v3.sql
```

**Size:** 547 lines  
**Neon Limit:** ~187 lines per execution  
**Result:** Cannot paste entire file into Neon SQL Editor at once

---

## ✅ Solutions

### Solution 1: Manual Splitting (Recommended for beginners)

📖 **Follow:** [NEON_MANUAL_IMPORT.md](NEON_MANUAL_IMPORT.md)

**Process:**
1. Open schema file in text editor
2. Copy Part 1 (lines 1-200) → Paste in Neon → Run
3. Copy Part 2 (lines 201-400) → Paste in Neon → Run
4. Copy Part 3 (lines 401-547) → Paste in Neon → Run
5. Verify tables created

**Time:** ~10 minutes  
**Difficulty:** Easy

---

### Solution 2: Automated Splitting (Recommended for developers)

```bash
cd database
node split-schema.js
```

**Output:**
- `part1-extensions-and-companies.sql` (~180 lines)
- `part2-users-and-auth.sql` (~180 lines)
- `part3-rfqs.sql` (~180 lines)
- `part4-quotations.sql` (~180 lines)
- `EXECUTION_GUIDE.md` (instructions)

Then execute each part in Neon SQL Editor in order.

**Time:** ~5 minutes  
**Difficulty:** Easy (requires Node.js)

---

### Solution 3: Use psql Command Line (Easiest - No splitting needed!)

```bash
cd RFQ_Buddy/Instructions

psql "postgresql://user:pass@host.neon.tech/db?sslmode=require" \
  -f rfq_tendering_platform_schema_v3.sql
```

Imports the entire schema at once!

**Time:** ~1 minute  
**Difficulty:** Easy (requires psql installed)

---

### Solution 4: Use Database Client (No limits)

**Tools:**
- DBeaver (free, cross-platform)
- pgAdmin (free, cross-platform)
- TablePlus (macOS/Windows)
- DataGrip (JetBrains)

**Process:**
1. Connect to Neon using connection string
2. Open SQL file
3. Execute all

**Time:** ~2 minutes  
**Difficulty:** Easy (requires database client)

## ❌ Common Errors

### "relation already exists"
**Cause:** Tables already created in a previous import  
**Fix:** Reset database and try again

👉 **See:** [RESET_DATABASE.md](RESET_DATABASE.md) for complete reset guide

**Quick Reset:**
```sql
DROP SCHEMA public CASCADE;
CREATE SCHEMA public;
GRANT ALL ON SCHEMA public TO neondb_owner;
```

### "syntax error near..."
**Cause:** Incomplete SQL statement copied  
**Fix:** Make sure you copy complete `CREATE TABLE ... ;` statements

### "line limit exceeded"
**Cause:** Copied too many lines at once  
**Fix:** Split into smaller chunks (use manual guide)
